Configure Google Drive Sync for DailiesApp

1 Get Google Cloud Credentials

You need to create OAuth 2.0 credentials in Google Cloud Console.

  1. Go to Google Cloud Console
  2. Create a new project or select existing one
  3. Enable "Google Drive API" in the API Library
  4. Go to "Credentials" → "Create Credentials" → "OAuth client ID"
  5. Application type: "Web application"

2 Configure OAuth Consent Screen

Set up the OAuth consent screen (required for testing).

// OAuth Consent Screen Configuration:
// 1. User Type: External
// 2. App Name: DailiesApp Test
// 3. User support email: your-email@gmail.com
// 4. Developer contact email: your-email@gmail.com
// 5. Scopes to add: ../auth/drive.file, ../auth/drive.appdata
// 6. Test users: Add your Google account email

3 Configure Authorized Domains

Add these authorized JavaScript origins and redirect URIs:

Authorized JavaScript origins:
• http://localhost
• http://localhost:8080 (if using different port)

Authorized redirect URIs:
• http://localhost/google-drive-callback.html
• http://localhost:8080/google-drive-callback.html

4 Copy Your Client ID

After creating credentials, copy your Client ID (looks like):

1234567890-abcdefghijklmnopqrstuvwxyz.apps.googleusercontent.com

Note: This is an OAuth Client ID, not an API Key. API Keys start with "AIzaSy".
